LuaSnip使用

2023年9月16日 · 阅读需 3 分钟

之前一直在网上搜来搜去，找不到如何在neovim上配置自己的代码片段，终于皇天不负有心人，找到了正确的配置方式。其实github上的官方文档已经很清楚了，可能是因为英文不好，所以一直没有看出来。特此记录下来，便于后用。

文件目录​

先贴一下文件目录，方便后续查看

~/.config/nvim/
├── lua
│   └── plugins
│       ├── luasnip.lua
├── snippets
│   ├── console.json
│   └── package.json

LazyVim中LuaSnip的配置​

重要变动是下面高亮的两行。两种配置方式，二者任选其一即可。
return {
  "L3MON4D3/LuaSnip",
  build = (not jit.os:find("Windows"))
      and "echo 'NOTE: jsregexp is optional, so not a big deal if it fails to build'; make install_jsregexp"
    or nil,
  dependencies = {
    "rafamadriz/friendly-snippets",
    config = function()
      -- load snippets/ to runtimepath.
      vim.opt.rtp = vim.opt.rtp + "~/.config/nvim/snippets"
      require("luasnip.loaders.from_vscode").lazy_load()
      -- require("luasnip.loaders.from_vscode").lazy_load({ paths = { "~/.config/nvim/snippets" } })
    end,
  },
  opts = {
    history = true,
    delete_check_events = "TextChanged",
  },
  -- stylua: ignore
  keys = {
    {
      "<tab>",
      function()
        return require("luasnip").jumpable(1) and "<Plug>luasnip-jump-next" or "<tab>"
      end,
      expr = true, silent = true, mode = "i",
    },
    { "<tab>", function() require("luasnip").jump(1) end, mode = "s" },
    { "<s-tab>", function() require("luasnip").jump(-1) end, mode = { "i", "s" } },
  },
}

配置snippets/package.json文件​

其实，在配置代码片段的时候，一直差的是这个文件。因为使用LuaSnip配置类似vscode配置的代码片段的时候是需要在代码片段文件所在的目录添加package.json文件的，用来指定代码片段所支持的语言等配置.

{
  "contributes": {
    "snippets": [
      {
        "language": ["all"],
        "path": "./console.json"
      }
    ]
  }
}

这里指定了console.json文件中些的代码片段支持所有语言。

配置snippets/console.json文件​

我这里比较简单，因为平时会些一些console.log，所以这里配置了比较容易识别是自己写的log的标志

{
  "print log": {
    "prefix": "cld",
    "body": [
      "console.log('dev: $1', $2);"
    ],
    "description": "console with dev keyword"
  }
}

这样，每次输入cld并选中后，就自动把console.log('dev: $1', $2)插入光标所在的位置。

  • $1表示插入代码片段后，光标所在的位置。
  • $2表示在$1位置输入自己想要的内容后，再按Tab键之后光标会在的位置

简单理解就是$1是光标的第一个位置，$2是光标的第二个位置。
