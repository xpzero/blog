使用Docusaurus构建博客

2023年9月23日 · 阅读需 7 分钟

一开始是用的hexo构建的博客，但是使用起来还是有些麻烦的。比如每篇文章的图片都要放在同一个目录下才能生效……；样式定义略显麻烦，只能在一个文件里使用css进行覆盖，等等，都让我觉得，hexo用起来不是那么的方便。

在做nvim的配置的时候，觉得LazyVim的官网挺好看的，看到了其页脚的Built with Docusaurus，就开始对Docusaurus一发不可收拾了。

为什么使用Docusaurus​

  1. 好看
  2. 图片可以和文章放在比较近的位置
  3. 除了大框架，其他内容可充分自定义
  4. 基于react，可以自定义组件及其样式
  5. 配置相对简单
  6. ……

这么好用为什么还不用？

安装​

官网描述的已经很清楚了，这里就不冗余了。

具体配置​

  • .env
  • package.json
  • docusaurus.config.js
  • custom.css
  • workflow.yml
这里使用了dotenv 。在部署的时候，将「.env」文件中的内容添加到了「process.env」中。「process.env」中的内容优先级高于「docusaurus.config.js」
# github 用户名
ORGANIZATION_NAME=xpzero

# 博客源码库所使用的分支
CURRENT_BRANCH=main

# 博客静态页面部署库的库名
PROJECT_NAME=blog

# 博客静态页面部署库所用的分支
DEPLOYMENT_BRANCH=main

# github 用户名 workflow中会用
GIT_USER=xpzero
# workflow中会用，密码要用personal key
# GIT_PASS=

# 将静态页面推送到部署库时的commit信息
CUSTOM_COMMIT_MESSAGE=publish post

通用组件​

可以在src/components中写通用组件。又分为全局可用的组件和局部引入的组件

全局使用​

全局组件即为，不需要手动引入即可使用的组件，就好像是内置组件一样。

配置步骤如下：

  • 添加src/theme/MDXComponents.js文件
  • 将src/components/中的某个组件通过MDXComponents文件导出

demo​

将官网的高亮组件设为全局使用

  • MDXComponents
  • Highlight
  • test.md
import React from "react";
import MDXComponents from "@theme-original/MDXComponents";
import Highlight from "@site/src/components/Highlight";

export default {
  ...MDXComponents,
  Highlight: Highlight,
};
import React from "react";

function Highlight({ children, color }) {
  return (
    <span
      style={{
        backgroundColor: color,
        borderRadius: "2px",
        color: "#fff",
        padding: "0.2rem",
      }}
    >
      {children}
    </span>
  );
}

export default Highlight;
