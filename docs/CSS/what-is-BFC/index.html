BFC是什么

常听说BFC这，BFC那的，也好几次的看过相关的文章，但碍不住总忘，这次就搞搞清楚，记录下来

BFC是什么​

全称Blocking Formatting Context, 块级格式化上下文，简称BFC,是一个独立于文档流的布局上下文，是一种使其内部的文本和元素不会影响到外部其他元素的排列方式。

BFC的特点​

  1. BFC内的浮动元素可以撑开父元素 => 包含内部浮动
  2. BFC的区域不会与相邻浮动元素重叠 => 清除外部浮动的影响
  3. BFC的区域可以避免与外部元素发生外边距折叠 => 阻止外边距折叠

BFC出现的条件​

  1. 根元素(html)
  2. float的值不为none
  3. position的值为absolute或fixed
  4. display为inline-block、inline-table、inline-flex、inline-grid、table、flex、grid、flow-root
  5. contain为layout、content、paint.

样例效果​

  • 撑开父元素的高度
  • 不与相邻浮动元素重叠(清除浮动)
  • 阻止外边距折叠
实时编辑器
function BFC() {
  const [height, setHeight] = useState(0);
  const boxStyle = {
    width: 100,
    height: 100,
    backgroundColor: "green",
    margin: 10,
  };
  const onClick = (e) => {
    const target = document.getElementById("target");
    target.style.overflow = "hidden";
    setHeight(target.offsetHeight);
  };

  return (
    <>
      <div style={{ backgroundColor: "skyblue" }} id="target">
        <div style={{ ...boxStyle, float: "left" }}>
          父元素现在的高度是{height}
        </div>
      </div>
      <br />
      <button onClick={onClick}>点我撑开</button>
    </>
  );
}

实时编辑器
function BFC() {
  const [bfcFlag, setBFCFlag] = useState(false);
  const [marginTop, setMarginTop] = useState(5);
  const [margin, setMargin] = useState(5);
  const boxStyle = {
    width: 100,
    height: 100,
    backgroundColor: "green",
    margin: 10,
  };

  const onClick = (_, number = 5) => {
    setMargin((v) => v + number);
    console.log("dev: number", number);
    setMarginTop((v) => (number ? v + number : v + 10));
  };

  useEffect(() => {
    if (bfcFlag) {
      onClick(null, 0);
    }
  }, [bfcFlag]);

  const createBFC = () => {
    const origin = document.getElementById("origin");
    origin.style.overflow = "hidden";
    setBFCFlag(true);
  };

  return (
    <>
      <div style={{ ...boxStyle }}>我的margin值是10px</div>
      <div id="origin">
        <div style={{ ...boxStyle, backgroundColor: "skyblue", margin }}>
          我的margin值是{margin}px，我与上面元素的距离是
          <span style={{ color: "red" }}>{marginTop}</span>px
        </div>
      </div>
      <button type="button" onClick={onClick}>
        第二个box的margin+5
      </button>
      <button onClick={createBFC}>为第二个box创建BFC</button>
    </>
  );
}

补充​

子元素margin-top失效​

当div元素嵌套时，如果父元素padding值为0，那么子元素的margin-top、margin-bottom就会「转移」为父元素的margin-top、margin-bottom

点击查看代码

        <div
          style={{

            height: 50,
            backgroundColor: "green",
            margin: 20,
          }}

}

解决方案​

1、在父层div加上：overflow:hidden；

2、把margin-top外边距改成padding-top内边距 ；

3、父元素产生边距重叠的边有不为 0 的 padding 或宽度不为 0 且 style 不为 none 的 border。

父层div加： padding-top: 1px;

4、让父元素生成一个 block formating context ，以下属性可以实现

  • float: left/right
  • position: absolute
  • display: inline-block/table-cell(或其他 table 类型)
  • overflow: hidden/auto

父层div加：position: absolute;
