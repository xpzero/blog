常用git命令

git常用命令备忘

提示

git版本：2.42.0

术语

  • Workspace：工作区/工作树
  • Index / Stage：索引/暂存区
  • Repository：仓库区（或本地仓库）
  • Remote：远程仓库

基本命令​

添加​

# 在当前目录初始化仓库
git init
# 添加变动到暂存区
git add <filepattern>
# 添加当前目录下的所有变动到暂存区
git add .
# 添加工作树中的所有变动到暂存区
git add -A

提交​

# 使用msg作为变动的提交信息
git commit -m <msg>
# 更新最新的一次提交「因为内容变化，需要强制推送」
git commit --amend <msg>
# 更新最新的一次提交，但不修改提交信息「因为内容变化，需要强制推送」
git commit --amend --no-edit
# 跳过pre-commit和commit-msg的git hooks
git commit -m <msg> -n
git commit -m <msg> --no-verify

暂存​

可以理解为临时存储

# 保存工作区和暂存区的所有变动
git stash save <name>
# 查看所有临时存储
git stash list
# 应用一个临时存储
git stash apply <stash>
# 应用一个临时存储并将其从list中移除
git stash pop
# 删除一个临时存储
git stash drop <stash>
# 删除所有临时存储
git stash clear

重置修改​

# 重置HEAD到commitId，并修改暂存区内容
git reset <commitId>
# 重置HEAD到commitId
git reset --soft <commitId>
# 重置HEAD到commitId，并修改暂存区与工作区中的内容
git reset --hard <commitId>

撤销修改​

# 恢复当前分支的工作区与暂存区到HEAD
git checkout .
# 撤回某次commit并提交新的commit
git revert <commitId>
# 执行时不打开默认编辑器，直接使用 Git 自动生成的提交信息。
git revert <commitId> --no-edit
# 只抵消暂存区和工作区的文件变化，不产生新的提交。
git revert <commitId> --no-commit

删除​

# 删除工作区文件，并且将这次删除放入暂存区
git rm <file>
# 停止追踪指定文件，但该文件会保留在工作区
git rm --cached <file>
# 改名文件，并且将这个改名放入暂存区
git mv [file-original] [file-renamed]
# 删除untracked files
git clean -f
# 连带untracked的目录一起删除
git clean -fd

分支​

# 查看本地所有分支
git branch
# 查看本地和远程所有分支
git branch -a
# 查看远程所有分支
git branch -r
# 基于当前分支新建一个分支并切换到新分支
git checkout -b <branch-name>
# 切换到之前的操作分支
git checkout -
# 删除本地分支
git branch -d <branch-name>
# 强制删除本地分支
git branch -D <branch-name>
# 删除远程分支
git push origin --delete <branch-name>
git branch -dr <remote/branch>

查看提交信息​

# 查看所有提交记录，并显示每次提交时变更的文件
git log --stat
# 显示过去5次的提交记录
git log -5 --pretty --oneline
# 显示文件的每次提交的diff
git log -p <file-name>
# 显示所有提交过的用户，按提交次数排序
git shortlog -sn
# 显示指定文件是什么人在什么时间修改过
git blame <file>
# 显示暂存区和工作区的差异
git diff
# 显示暂存区和上一个commit的差异
git diff --cached <file>

合并​

# 将<branch-name>合并到当前分支
git merge <branch-name>
# 合并<commitId>到当前分支
git cherry-pick <commitId>
