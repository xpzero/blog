如何让antd中的InputNumber组件只能输入数字？

因为antd 的InputNumber 组件最终的输入内容只能在失焦后看到，但是有时候用户想要在输入的时候就能看到最终的效果。例如，如果功能设计为InputNumber 只允许输入整数，且非数字内容不允许输入。因为带了一些业务逻辑，这时候antd 的InputNumber 自身的功能就有些不够用了。所以，记录下自己写的一个关于antd 的InputNumber 不允许输入小数点的demo。

实现效果​

  • 只能输入数字，非数字不允许输入
  • 实时显示最终的输入结果
  • 不超过最大值，不小于最小值

一个InputNumber实现​

import React, { useEffect, useRef, useState } from "react";
import { InputNumber, InputNumberProps } from "antd";
export default function IntegerInputNumber(props: InputNumberProps) {
  const { defaultValue, min, max, step } = props;
  const [value, setValue] = useState(defaultValue);
  const [forceRefresh, setForceRefresh] = useState(0);

  const elemRef = useRef<HTMLInputElement>(null);

  useEffect(() => {
    if (forceRefresh) {
      elemRef.current?.focus();
    }
  }, [forceRefresh, value]);

  const parser = (newValue: string) => {
    console.log("dev: parser-value", newValue);
    if (newValue === `${value}`) {
      return;
    }
    if (newValue === "") {
      setValue("");
      return;
    }
    // 这里拿到的数字可能和value一样
    const num = parseInt(newValue);
    if (num === value) {
      // It will lead input's blur.
      setForceRefresh((v) => v + 1);
      return;
    }

    let realNum = num as number;
    if (/\D/.test(newValue)) {
      // 非数字字符串，直接结束函数，仍然使用之前的value
      if (Number.isNaN(num)) {
        return;
      }

      if (typeof min === "number") {
        if (num < min) {
          realNum = min;
        }
      }

      if (typeof max === "number") {
        if (num > max) {
          realNum = max;
        }
      }
    }
    // It will lead to input's blur.
    setValue(realNum);
  };

  return (
    <InputNumber
      ref={elemRef}
      key={forceRefresh}
      parser={parser}
      value={value}
      min={min}
      max={max}
      step={step}
    />
  );
}

上面的代码实现了禁止antd 的InputNumber 组件输入非数字字符的功能。从代码上可以看出，是通过控制组件的value 的方式实现结果的动态展示。因为setState 之后会失焦，所以监听forceRefresh 和value 的值，并重新聚焦。

但是上面的功能还不太完善，可以试试右边的两个上下箭头的功能，会发现他们被点击后没有任何效果。因为value 现在是手动控制的，所以即使内置的onStep 有认真执行，但也无济于事。下面给出了处理上下箭头功能的完整代码。

处理上下箭头的功能​

import React, { useEffect, useRef, useState } from "react";
import { InputNumber, InputNumberProps } from "antd";
export default function IntegerInputNumber(props: InputNumberProps) {
  const { defaultValue, min, max, step } = props;
  const [forceRefresh, setForceRefresh] = useState(0);
  const valueRef = useRef(defaultValue);

  const elemRef = useRef<HTMLInputElement>(null);

  useEffect(() => {
    if (forceRefresh) {
      elemRef.current?.focus();
    }
  }, [forceRefresh]);

  const parser = (newValue: string) => {
    console.log("dev: parser-value", newValue);
    if (newValue === `${valueRef.current}`) {
      return;
    }
    if (newValue === "") {
      valueRef.current = "";
      return;
    }
    // 这里拿到的数字可能和value一样
    const num = parseInt(newValue);
    if (num !== valueRef.current) {
      let realNum = num as number;
      if (/\D/.test(newValue)) {
        // 非数字字符串，直接结束函数，仍然使用之前的value
        if (Number.isNaN(num)) {
          return;
        }

        if (typeof min === "number") {
          if (num < min) {
            realNum = min;
          }
        }

        if (typeof max === "number") {
          if (num > max) {
            realNum = max;
          }
        }
      }
      valueRef.current = realNum;
    }

    // It will lead input's blur.
    setForceRefresh((v) => v + 1);
  };

  const onMyStep = (value: number) => {
    valueRef.current = value;
    setForceRefresh((v) => v + 1);
  };
  return (
    <InputNumber
      ref={elemRef}
      key={forceRefresh}
      parser={parser}
      value={valueRef.current}
      min={min}
      max={max}
      step={step}
      onStep={onMyStep}
    />
  );
}

在onStep 直接修改value 值会导致下面的错误。

根据报错可以看出是因为重复调用了setState 导致的，所以把value 从state 改成了ref 这样就不会重复调用setState了。所以现在InputNumber 的值由valueRef 控制

添加外部onChange事件的处理​

可能开发过程中需要在InputNumber 的值发生变化后进行一些其他的处理。

但是也不能直接写在组件里，因为组件被使用的地方可能不止一处，所以就需要将onChange 事件传递进组件使用

不过，还需要考虑另一件事。那就是，什么时候触发这个onChange 事件呢？如果实时触发，且onChange 事件中的处理较繁重就会影响页面渲染效果。

换种思考方向，一般什么时候使用InputNumber 呢？大多是要和form 表单一起的，InputNumber 只是其中的一项。而form 表单提交的时候，InputNumber 基本都是blur 状态。所以这里就将onChange 事件的触发放在InputNumber 组件的onBlur 事件中，且仅当valueRef 的值有变化之后才触发，否则即使触发了也没有任何意义。当然，也有额外情况，但不在本文范围内。

import React, { useEffect, useRef, useState } from "react";
import { InputNumber, InputNumberProps } from "antd";
export default function IntegerInputNumber(props: InputNumberProps) {
  const { defaultValue, min, max, step, onChange } = props;
  const [forceRefresh, setForceRefresh] = useState(0);
  const valueRef = useRef(defaultValue);

  const elemRef = useRef<HTMLInputElement>(null);

  useEffect(() => {
    if (forceRefresh) {
      elemRef.current?.focus();
    }
  }, [forceRefresh]);

  const parser = (newValue: string) => {
    console.log("dev: parser-value", newValue);
    if (newValue === `${valueRef.current}`) {
      return;
    }
    if (newValue === "") {
      valueRef.current = "";
      return;
    }
    // 这里拿到的数字可能和value一样
    const num = parseInt(newValue);
    if (num !== valueRef.current) {
      let realNum = num as number;
      if (/\D/.test(newValue)) {
        // 非数字字符串，直接结束函数，仍然使用之前的value
        if (Number.isNaN(num)) {
          return;
        }

        if (typeof min === "number") {
          if (num < min) {
            realNum = min;
          }
        }

        if (typeof max === "number") {
          if (num > max) {
            realNum = max;
          }
        }
      }
      valueRef.current = realNum;
    }

    // It will lead input's blur.
    setForceRefresh((v) => v + 1);
  };

  const onMyStep = (value: number) => {
    valueRef.current = value;
    setForceRefresh((v) => v + 1);
  };

  const onMyBlur = () => {
    if (valueRef.current !== defaultValue) {
      onChange?.(valueRef.current as number | string);
    }
  };

  return (
    <InputNumber
      ref={elemRef}
      key={forceRefresh}
      parser={parser}
      value={valueRef.current}
      min={min}
      max={max}
      step={step}
      onStep={onMyStep}
      onBlur={onMyBlur}
    />
  );
}

再次优化​

其实上面的代码还是有一些缺陷。如果在数字中间输入非数字字符，则当前位置之后的数字都会因为parseInt 而消失。下面是参考Semi-ui 的正则做出的调整。

import React, { useEffect, useRef, useState } from "react";
import { InputNumber, InputNumberProps } from "antd";
export default function IntegerInputNumber(props: InputNumberProps) {
  const { defaultValue, min, max, step, onChange } = props;
  const [forceRefresh, setForceRefresh] = useState(0);
  const valueRef = useRef(defaultValue);

  const elemRef = useRef<HTMLInputElement>(null);

  useEffect(() => {
    if (forceRefresh) {
      elemRef.current?.focus();
    }
  }, [forceRefresh]);

  const parser = (newValue: string) => {
    console.log("dev: parser-value", newValue);
    if (newValue === `${valueRef.current}`) {
      return;
    }
    if (newValue === "") {
      valueRef.current = "";
      return;
    }
    // 这里拿到的数字可能和value一样
    const realValue = newValue.replace(/\D/g, "");
    if (realValue !== valueRef.current) {
      let realNum = +realValue;
      if (/\D/.test(newValue)) {
        if (typeof min === "number") {
          if (realNum < min) {
            realNum = min;
          }
        }

        if (typeof max === "number") {
          if (realNum > max) {
            realNum = max;
          }
        }
      }
      valueRef.current = realNum;
    }

    // It will lead input's blur.
    setForceRefresh((v) => v + 1);
  };

  const onMyStep = (value: number) => {
    valueRef.current = value;
    setForceRefresh((v) => v + 1);
  };

  const onMyBlur = () => {
    if (valueRef.current !== defaultValue) {
      onChange?.(valueRef.current as number | string);
    }
  };

  return (
    <InputNumber
      ref={elemRef}
      key={forceRefresh}
      parser={parser}
      value={valueRef.current}
      min={min}
      max={max}
      step={step}
      onStep={onMyStep}
      onBlur={onMyBlur}
    />
  );
}
